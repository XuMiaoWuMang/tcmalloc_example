# tcmalloc_example
一个C++的高性能内存池

## 功能

学习tcmalloc的实现原理，实现一个简单的内存池

## 使用方式

使用 ConcurrentAlloc 和 ConcurrentFree 函数申请和释放内存块

```c++
#include "ConcurrentAlloc.h"

int main()
{
    void* ptr = ConcurrentAlloc(1024);
    ConcurrentFree(ptr, 1024);
    return 0;
}
```
如果想看看程序效果，可以修改 test.cpp 文件，然后在命令行输入以下命令：
```bash
make && ./main
```

## 学习心得与记录

tcmalloc是一个三层架构的内存池，分别是：

1. 线程缓存（ThreadCache）：每个线程都有自己的线程缓存，用于快速分配和释放小对象，不存在锁的竞争。

2. 中央缓存（CentralCache）：用于管理线程缓存之间的内存分配和释放，负责从页缓存中获取大块内存，并将其切割成小块，分配给线程缓存。

3. 页缓存（PageCache）：用于管理以页为单位的大块内存的分配和释放，负责从操作系统获取大块内存，并将其划分为页。

### 线程缓存（ThreadCache）

使用TLS确保每个线程都有自己的线程缓存，这样就不存在锁的竞争，可以快速分配和释放小对象。如果需要获取大块内存，会直接从页缓存中获取大块内存，不需要将其切割成小块，直接分配给线程缓存。

> 为什么不需要切分成小块？
> 最主要的原因是：如果直接需要从页缓存获取大块内存的基础页依旧超过了页缓存的最大页数，页缓存已经无法管理对应的内存块，只能从操作系统获取大块内存，但是由于这个内存已经很大了，如果多申请内存，会导致很严重的内碎片问题。
> 毕竟此时已经是以 1MB 为单位的内存块。

每个线程的线程缓存使用TLS隔绝，确保每个线程的线程缓存是独立的，互不干扰。使用 _maxSize 成员变量保存单个自由链表的上限，即是从控制缓存中批量申请的依据，也是释放给控制缓存的依据。

使用链表数组管理线程缓存中的所有内存块，相同大小的内存块都有一个指针指向下一个内存块，可以对不同大小的内存块实现快速的分配和释放。

### 中央缓存（CentralCache）

使用哈希桶管理 SpanLists ，每个 SpanList 管理所有相同大小的 Span ，每个 Span 管理最小为1页的内存块，当分配给线程缓存时，按照 线程缓存的 `_maxSize` 成员变量 的上限，从 SpanList 中分配内存块。

使用 _useCount 管理分配给线程缓存的内存块数量，当 _useCount 达到0的时候，说明所有分配给线程缓存的内存块都被释放了，可以将SpanList中的Span还给页缓存中。

但是，不能只用 _useCount 来判断是否可以将 SpanList 中的 Span 还给页缓存中，因为 Span 可能才刚从页缓存中获取，还没有被分配给线程缓存，此时 _useCount 为0，所以使用 _inUsed 来判断是否可以将 SpanList 中的 Span还给页缓存中。

### 页缓存（PageCache）

使用哈希桶管理页缓存中的所有 Span ，依据 Span 所管理的页数作为 SpanList 的索引，每个 SpanList 管理所有相同大小的 Span 。

这样可以快速地根据页数来获取对应的 SpanList ，当没有对应的 SpanList 时，会向系统申请一个最大页数的内存块，并将其维护起来。

通过递归的方式进行内存复用，如果刚好是控制缓存需要的，就直接返回，如果不是，就切割一部分给控制缓存，剩下的依据页数维护起来。

使用哈希表管理 _page_id 和 Span* 的映射关系，快速地根据 _page_id 来获取对应的 Span* 。

使用 _page_id 作为 Span 的唯一标识，_page_id 是起始地址的页号，使用 `(ptr >> PAGE_SHIFT)` 来获取。PAGE_SHIFT 是页大小的位移，例如 1页=4MB ，则 PAGE_SHIFT=12 。

如果中心缓存中还回来了Span，则需要依据 _page_id 和 _isUsed 来判断相邻 Span 是否可以合并，同时需要维护哈希表中的映射关系，来判断相邻 Span 是否可以合并，同时需要维护哈希表中的映射关系。

值得注意的是，当线程缓存需要一个超过 1MB 的内存块时，只需要维护该内存块的首页和 Span* 的映射关系，其他页的映射关系不需要维护。因为 PageCache 无法管理超过 1MB 的内存块。所以不存在合并的可能性，不许要在合并是考虑合并该 Span 。

> 如果合并之后的 Span 的页数可能大于哈希桶的最大索引，直接不合并。

对于高频的内存申请请求和释放请求，这种模式由不存在锁竞争，可以快速分配和释放内存块。同时对申请的内存块大小使用分级设计，控制内碎片的占比，缓解内存碎片问题。
